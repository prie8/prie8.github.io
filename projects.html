Personalized Diabetic Wound Healing

A CRISPR-Based Synthetic Biology Approach Using Engineered Probiotics

Introduction & Rationale

My research addresses the critical failure of current antibiotic treatments for Diabetic Foot Ulcers (DFUs), which are complicated by chronic inflammation and Staphylococcus aureus biofilms. Witnessing my grandfather’s struggle with diabetic complications, I hypothesized that a static chemical treatment is insufficient for such a dynamic biological problem.

I proposed a "living medicine": a commensal E. coli engineered to sense the specific microenvironment of a diabetic wound (acidic pH < 6.0 and hyperglycemia> 8.0 mM) and respond by secreting therapeutic agents.

Biological Circuit Design

I designated a dual-input genetic logic gate using the pUC19 plasmid backbone:

  • Input 1 (Acidity): The PcadA promoter activates in pH < 6.0, driving expression of Bovine DNase I to degrade the biofilm matrix.
  • Input 2 (Hyperglycemia): The PtsG/Crp system detects glucose > 8.0 mM, triggering a CRISPR activation (CRISPRa) system (dCas9-VP64).
  • Output: The CRISPRa system targets the PlacZ promoter to upregulate Human Interleukin-10 (IL-10), an anti-inflammatory cytokine.

Computational Implementation: PhysiCell

I validated this system through high-fidelity agent-based modeling using PhysiCell. This was the most technically demanding phase, involving the customization of the framework to mimic synthetic biological functions rather than standard tissue mechanics.

1. Customizing the Phenotype

The standard PhysiCell library models immune cells consuming pathogens via phagocytosis. However, my engineered bacteria needed to secrete an enzyme (DNase I) to damage the biofilm externally. I wrote a custom C++ phenotype function, predator_phenotype_function, to replace the default interaction model.

// Conceptual Logic for Damage Induction
if( local_glucose > glucose_threshold && current_pH < ph_threshold ) {
  double damage = damage_rate * dt;
  std::vector<Cell*> neighbors = get_possible_neighbors(pCell);
  for( Cell* pNeighbor : neighbors ) {
    if( pNeighbor->type == biofilm_cell_type_index ) {
      pNeighbor->phenotype.damage += damage; // Enzymatic degradation
    }
  }
}

2. XML-Based Boolean Networks

To encode the genetic circuit without hardcoding every parameter in C++, I utilized PhysiCell’s XML-based Boolean Network feature. This allowed me to define the "brain" of the smart bacteria directly in the configuration settings:

  • Nodes: Defined is_low_pH and is_high_glucose as boolean states.
  • Logic: Mapped thresholds (pH < 6.0) to these states.
  • Actuation: Linked is_low_pH == true to the secretion rate of DNase I.

Safety Modeling: NetLogo & Statistics

To address bioethics, I modeled a thermal kill switch in NetLogo. The simulation tested a population of bacteria "escaping" from the cool wound site (30-35°C) to the core body (>37°C).

Running 100 stochastic simulations (n=100) using BehaviorSpace, the model predicted a mean effectiveness of >97%.

Statistical Rigor

With a mean survival rate of 3% and an approximate standard deviation of 1.2%, I calculated the 95% Confidence Interval to be [2.76%, 3.24%]. This narrow range confirmed the high reliability of the biocontainment mechanism in silico.

Data Analysis Pipeline

The simulation generated gigabytes of data over a virtual 120-hour period. I developed a Python pipeline using Pandas and Matplotlib to:

  • Parse binary .mat files generated by PhysiCell.
  • Track cell population counts (Probiotic vs. S. aureus).
  • Visualize the "Cell States" over time (e.g., how many bacteria were secreting IL-10 vs. DNase I).

Result: The "Smart Bacteria" system reduced biofilm mass by 50-70%, significantly outperforming the antibiotic benchmark (~30%) in this coupled simulation.
